## 策略概述

本策略是基于 **Backtrader** 框架开发的多因子趋势跟踪策略，结合了均线系统、价格突破和动态止损机制。

### 主要特征
- **时间周期**: 5分钟K线（由1分钟数据压缩生成）
- **交易品种**: 期货合约（示例使用橡胶SHFE.ru2509）
- **策略类型**: 趋势跟踪 + 均值回归混合策略
- **资金管理**: 百分比仓位管理（98%资金利用率）

## 核心逻辑

### 1. 技术指标体系

| 指标名称 | 计算方式 | 用途 |
|---------|---------|------|
| SMA_K (快均线) | 11周期简单移动平均 | 短期趋势判断 |
| SMA_M (慢均线) | 350周期简单移动平均 | 中期趋势判断 |
| SMA_J (趋势线) | 598周期简单移动平均 | 长期趋势过滤 |
| Highest | 120周期最高价 | 阻力位参考 |
| Lowest | 120周期最低价 | 支撑位参考 |

### 2. 开仓条件

#### 多头开仓（满足以下所有条件）
```python
1. 当前无持仓
2. 最高价 > SMA_J（长期趋势向上）
3. |SMA_M[-5] - SMA_M[0]| > 0.9（慢均线有足够斜率）
4. 满足以下任一突破信号：
   - 6根K线前最高价 < SMA_M 且 当前最高价 > SMA_M
   - SMA_K上穿SMA_M（金叉）
   - SMA_M上穿SMA_J
```

#### 空头开仓（满足以下所有条件）
```python
1. 当前无持仓
2. 最低价 < SMA_J（长期趋势向下）
3. |SMA_M[-5] - SMA_M[0]| > 0.9（慢均线有足够斜率）
4. 满足以下任一突破信号：
   - 6根K线前最高价 > SMA_M 且 当前最高价 < SMA_M
   - SMA_K下穿SMA_M（死叉）
   - SMA_M下穿SMA_J
```

### 3. 平仓/止损逻辑

#### 多单平仓条件（满足任一即平仓）
```python
1. 收盘价 < 动态止损价 - 10点缓冲
2. 收盘价 < SMA_M（跌破慢均线）
```

#### 空单平仓条件（满足任一即平仓）
```python
1. 收盘价 > 动态止损价 + 10点缓冲
2. 收盘价 > SMA_M（突破慢均线）
```

### 4. 动态止损算法（zhisun函数）

**多单止损计算**:
- 向前搜索最多200根K线
- 寻找连续两根"阴线转阳线"的K线组合
- 取这两根K线的最低价中的较小值作为止损位

**空单止损计算**:
- 向前搜索最多200根K线
- 寻找连续两根"阳线转阴线"的K线组合
- 取这两根K线的最高价中的较大值作为止损位

**设计意图**: 基于近期价格结构的转折点设置止损，避免被正常波动触发。

## 参数配置

### 策略参数
```python
params = (
    ('LowHi_price', 120),  # 最高最低价计算周期
)
```

### 经纪人参数
```python
初始资金: 100,000元
交易佣金: 0.04% (万分之四)
杠杆倍数: 1倍
仓位比例: 98%
```

### 可调整参数建议

| 参数位置 | 当前值 | 可调范围 | 影响说明 |
|---------|-------|---------|---------|
| `timeperiod=11` (SMA_K) | 11 | 5-30 | 越小越敏感，越大越平滑 |
| `timeperiod=350` (SMA_M) | 350 | 100-500 | 中期趋势判断核心参数 |
| `timeperiod=598` (SMA_J) | 598 | 300-800 | 长期趋势过滤阈值 |
| `LowHi_price=120` | 120 | 60-200 | 高低点统计周期 |
| `abs差值>0.9` | 0.9 | 0.5-2.0 | 均线斜率敏感度 |
| `止损缓冲±10` | 10 | 5-20 | 防止假突破的缓冲空间 |
| `commission=0.0004` | 0.04% | 根据实际调整 | 不同品种佣金不同 |

## 注意事项

### ⚠️ 重要提醒

1. **数据质量**
   - 确保CSV文件无缺失值
   - 检查时间戳连续性
   - 验证价格数据合理性（无负值、异常跳空）

2. **过拟合风险**
   - 当前参数针对特定品种优化
   - 跨品种使用前需重新测试
   - 建议在多个市场周期验证

3. **滑点与执行**
   - 回测假设理想成交
   - 实盘需考虑滑点成本
   - 大单可能冲击市场价格

4. **性能优化**
   - `zhisun()`函数每次调用遍历最多200根K线
   - 高频数据回测速度较慢
   - 可考虑向量化优化

5. **风险管理**
   - 98%仓位比例较高，建议根据风险承受能力调整
   - 单一策略不宜投入全部资金
   - 建议配合其他策略分散风险

### 🔧 常见问题

**Q: 回测结果为空或报错？**
- 检查数据文件路径是否正确
- 确认CSV列名与代码一致
- 验证时间范围是否有效

**Q: 交易次数过少？**
- 降低均线周期参数
- 减小`abs差值>0.9`阈值
- 放宽开仓条件

**Q: 胜率过低但盈利？**
- 正常现象，趋势策略特点
- 关注盈亏比而非胜率
- 检查止损是否过于宽松

**Q: 如何导出交易记录？**
- 在`notify_trade()`中添加写入CSV逻辑
- 或使用`cerebro.addwriter(bt.WriterFile)`
